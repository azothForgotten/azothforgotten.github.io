Minesweeper implementation:

This Minesweeper game is totally implemented in JavaScript.

Below is a breakdown of the functionality that meets the requirements for this project. Each game other than the XML generated one is randomized.

Requirements:

  1. Game grid that uses CSS and is generated by JavaScript.
    • View.js uses generateGrid() to create the HTML table and assign style classes to the cells.
  2. User input in the form of text, select options, and/or buttons.
    • gamesettings.html has a few different inputs from the user:
      • A select field allows the user to select preset game sizes.
      • Two input fields allow the user to customize game sizes.
      • A button saves the user's game size inputs into localStorage for use on randomgame.html.
  3. User input in the form of mouse clicks:
    • View.js uses addClickHandler() to create event handlers for when a cell is clicked or right-clicked.
  4. Dynamic modification of HTML using innerHTML or DOM functions:
    • View.js uses refreshModel() to update style classes and innerHTML based on the model.
  5. Use of XMLHttpRequest to load JSON or XML game data that is displayed on the grid page:
    • game.xml is a preset game.
    • View.js uses createGridfromXML() to call an XMLHttpRequest and generate a game based on XML data.
    • For simplicity this can be seen on xmlgame.html rather than the normal randomgame.html.
  6. Incorporates HTML 5 audio tags, video tags, canvas, or local storage:
    • randomgame.html and xmlgame.html contain an audio tag which is played when a cell containing a bomb is clicked.
  7. Game logic code:
    • gameGridObj() stores information about the current game.
      • Properties:
        • x,y: The length of each side.
        • bombs: Number of bombs
        • flags: Number of flags in place.
        • isOver: Boolean to signify whether not the game has ended.
        • cellArray: An array of cellObj objects.
        • sound: Boolean to signify if a sound has been played.
      • Methods:
        • createGridArray() initializes cellObj array and creates the correct number of cellObj.
        • getCellArray() returns this gameGridObj's cellArray property.
        • getCell(x,y) returns the cellObj for the given x and y indices.
        • getFlags() returns the flags property.
        • getBombs() returns the bombs property.
        • soundPlayed() checks the sound property, if a sound hasn't been played, sets sound to true and returns false, otherwise returns the sound property.
        • checkGameProgress() returns true if a bomb has been clicked or if all cells other than bombs are clicked and all bombs are flagged, otherwise returns false.
        • checkWin() returns false if a bomb has been clicked, otherwise returns true.
        • plantBombs() randomly assigns the correct number of bombs to this gameGridObj's cellObj's.
        • getAdjacentCells(x,y) returns an array of cellObj adjacent to the cell at the given indices.
        • addFlag() increments the gameGridObj's flags property.
        • removeFlag() decrements the gameGridObj's flags property.

    • cellObj() stores infromation about the cell.
      • Properties:
        • x,y: coordinates to this cell.
        • isFlagged: Boolean to signify if this cell has been flagged.
        • isBomb: Boolean to signify if this cell is a bomb.
        • isClicked: Boolean to signify whether not this cell has been clicked.
        • adjacentBombs: Number of adjacentBombs.
        • gameGrid: The parent gameGridObj.
      • Methods:
        • makeBomb() sets isBomb property true.
        • unMakeBomb() sets isBomb property false.
        • setAdjacent(bombs) sets the adjacentBombs property to the passed in "bombs" value.
        • addBomb() increments the adjacentBombs property.
        • flagMe() if the current cellObj's isFlagged property is true set it to false, or vice versa.
        • clickMe() recursive function to click cells. Clicks only one if the cell has adjacent bombs or is a bomb. Otherwise recursively click until the cell has an adjacentBombs value.
  8. Extra Credit: Game uses localStorage to store user settings from gamesettings.html and retrieve them on randomgame.html. (Number of bombs and Number of cells per side).
